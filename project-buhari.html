Project Description

A web-based restaurant ordering application designed for Buhari Hotel that streamlines the ordering process for waiters and provides comprehensive business reporting capabilities. The system allows efficient management of customer orders with built-in business rules enforcement.

Key Features

  • Menu Browsing: Categorized menu system with Main Dish, Side Dish, and Dessert sections
  • Order Management: Create and manage customer orders with real-time updates
  • Business Rules: Automated enforcement of ordering rules and constraints
  • Order History: Complete tracking and history of all orders
  • Business Reports: Comprehensive reporting dashboard for insights and analytics
  • Waiter Interface: Optimized UI for restaurant staff efficiency

Technology Stack

Frontend:

  • React 19.x with TypeScript for type-safe component development
  • Vite for fast build tooling and hot module replacement
  • Tailwind CSS 4.x for utility-first styling
  • Shadcn UI and Radix UI for accessible component library
  • React Router for client-side routing
  • Recharts for data visualization in reports

Backend:

  • Node.js with Express.js 5.x for RESTful API
  • TypeScript 5.x for type safety across the stack
  • PostgreSQL for reliable relational database
  • Prisma ORM 6.x for type-safe database queries
  • Supabase for authentication and real-time capabilities

Technical Highlights

  • Full TypeScript implementation across frontend and backend
  • Modern React 19 features and best practices
  • Type-safe database operations with Prisma ORM
  • Responsive design with Tailwind CSS
  • Real-time order updates using Supabase
  • Component-driven architecture with Shadcn UI
